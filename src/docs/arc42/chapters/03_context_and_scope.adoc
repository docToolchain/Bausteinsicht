== Technical context

* *Inputs*: model `*.jsonc`, draw.io `*.drawio`, sync state
  `.bausteinsicht-sync`, optional `template.drawio`.
* *Outputs*: same files (mutated atomically) plus exports
  (PNG/SVG/PUML/MMD/ADOC/MD).
* *External processes*: only the draw.io CLI, only when the user runs
  `bausteinsicht export` _(`internal/export/export.go:32-45`)_.
* *Network*: none. Bausteinsicht does not open sockets, make HTTP
  calls, or rely on TLS _(OQ-7 team answer; `README.adoc:194-196`)_.

== Trust boundaries

_(OQ-7 team answer)_

[cols="1,3", options="header"]
|===
| Boundary | Notes

| *Fully trusted* — CLI flags
| The user types them directly; behaviour mirrors `cat`/`jq`/`sed`. No
sandboxing.

| *Semi-trusted* — model and template files
| May come from a shared Git repository. Parsed and validated, but the
process is not sandboxed. `LoadDocument` validates structural
integrity before any modification _(`internal/drawio/document.go:30-49`)_.

| *Untrusted* — agent / CI environment
| Standard sandboxing (container, restricted directory, read-only
mounts) is the caller's responsibility. SEC-001 / SEC-016 reject
`..` in paths supplied via `--model`, `--template`, `--output`
_(`cmd/bausteinsicht/root.go:78-89`,
`cmd/bausteinsicht/export.go:46-50`)_.
|===

== Out of scope

_(OQ-2 + OQ-3 team answers)_

* Multi-user real-time editing.
* Server / web UI.
* As-Is/To-Be modelling, custom validation rules, polished PNG/SVG
  pipeline integrations, CI gating actions, Structurizr/LikeC4
  importers — these are *planned future considerations* but not
  delivered in v1.
