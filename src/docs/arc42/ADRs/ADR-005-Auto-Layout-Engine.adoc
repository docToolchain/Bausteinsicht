== Context

When forward sync places elements on a fresh page, the placement
must be:

* deterministic (same model → same positions),
* readable (kinds form recognisable rows; alphabetic order within tier),
* portable (no system-installed graph layouter),
* fast (sync budget is < 100 ms for 200 elements, OQ-8).

Candidates: *Custom layered/grid algorithm in pure Go*, *GraphViz
(dot) shell-out*, *ELK.js / d3-dag (Node)*, *Dagre (JS)*, *Cytoscape*.

== Decision

*Custom layered/grid algorithm in pure Go*. Three layouts are
exposed: `layered` (default), `grid`, and `none`
_(`internal/sync/layout.go:42-60`,
`internal/model/validate.go:167-172`)_.

== Rationale

. *Zero runtime dependencies.* Bausteinsicht ships as a single static
  binary; depending on GraphViz or a JS engine would break the
  zero-dependency guarantee from PRD-001 §8.
. *Determinism is a hard requirement.* The layered algorithm groups by
  kind (in `specification.elements` order), then sorts alphabetically
  within tier _(`internal/sync/layout.go:62-80`,
  test `TestComputeLayout_Layered_Deterministic`)_.
. *Fast.* No subprocess fork, no JSON marshal/parse, no temp files.
. *Tractable.* The product currently supports up to a few hundred
  elements per view; advanced graph layouters bring complexity that
  is not yet justified.
. *User-controllable.* Users who dislike auto-layout can set
  `"layout": "none"` or simply move shapes in draw.io — reverse sync
  preserves positions.

== Consequences

* Three layout strategies are supported; users opt out via
  `"layout": "none"` _(`internal/model/validate.go:167-172`)_.
* New elements get a red dashed border so the user can spot and
  reposition them _(`internal/sync/forward.go:13-15`,
  test `TestApplyForward_NewElementVisualMarker`)_.
* Reverse sync preserves user-edited positions; the layout engine is
  invoked only when (a) a page is brand new, or (b) `--relayout` is
  passed _(`internal/sync/forward.go:135-137`)_.
* If a future view contains thousands of elements, the algorithm may
  need revision (open consideration, not a current limit).
