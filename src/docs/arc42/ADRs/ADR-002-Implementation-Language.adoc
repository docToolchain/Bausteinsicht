== Context

The CLI must:

* Produce a single static binary across linux, darwin, windows on
  amd64 and arm64 with no runtime dependencies _(`Makefile:21-46`,
  `.goreleaser.yml:14-25`)_.
* Achieve cold-start time < 10 ms and sync time < 100 ms for ≤ 200
  elements _(OQ-8 team answer)_.
* Integrate with draw.io XML parsing without bundling a JVM, Node, or
  Python runtime.

Candidates considered: *Go*, *Rust*, *TypeScript (Node)*, *Java
(GraalVM native)*, *Python*.

== Decision

*Go (>= 1.25)*.

== Rationale

. *Single static binary, no runtime deps.* `CGO_ENABLED=0` produces a
  fully static executable per OS/arch _(`Makefile:23-46`,
  `.goreleaser.yml:18-19`)_.
. *Cross-compilation for free.* Six target binaries built from one
  developer machine without docker or QEMU
  _(`Makefile:21-46`).
. *Fast cold start.* Native binary, no VM warm-up — meets the
  10 ms budget _(OQ-8)_.
. *Strong CLI/XML/file-watching ecosystem.* `spf13/cobra`,
  `beevik/etree`, `fsnotify/fsnotify` are mature with small APIs.
. *Mature security tooling.* `gosec`, `govulncheck`, `nilaway`,
  `staticcheck` are first-class for Go _(`Makefile:60-90`)_.
